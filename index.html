JDK en Rust desde cero

En desarrollo

Reimplementación del núcleo de la plataforma Java en Rust. Está organizado en componentes independientes:

  • Listo
    Compilador (javac) javac propio que produce bytecode JVM a partir de código Java.
  • Listo
    JVM — ejecución de bytecode Máquina virtual que ejecuta bytecode real, incluyendo programas no triviales.
  • Listo
    GC mark-compact Recolector de basura propio de tipo mark-compact.
  • Listo
    javap — 99% de paridad Herramienta javap con 99% de paridad de salida frente a OpenJDK 25.
  • En desarrollo
    Concurrencia — threads y GC multihilo Modelo de concurrencia y recolección de basura en entornos multihilo.
  • En desarrollo
    Demás tools (jdb, etc.) Resto del toolchain (depurador jdb y otras herramientas) y verificación de compatibilidad con Java 25.

Las librerías estándar provienen de OpenJDK (no reimplementadas). El núcleo —compilador, JVM y GC— es de desarrollo propio.

BoxBreaker (BBK)

Plugin publicado

Compilador de RPG (IBM i / AS/400) que traduce el código a un lenguaje intermedio propio (BBK), con la meta de hacer RPG portable fuera del hardware IBM. Está organizado en componentes independientes:

  • Listo
    Front-end RPG → BBK Lexer, parser y AST que compilan código RPG al IR propio (BBK). Cubierto por 63 tests.
  • Listo
    Backend JVM Genera bytecode JVM ejecutable a partir de BBK usando la librería ASM.
  • Listo
    Backend C Transpila BBK a C99. Cubre aproximadamente el 85% del lenguaje.
  • Publicado
    Plugin de IntelliJ Publicado en JetBrains Marketplace (v1.0.0, 72 tests): highlighting, autocompletado type-aware, inspecciones, navegación cross-file y refactoring. Ver en JetBrains Marketplace ↗
  • En desarrollo
    Runtime IBM i Capa de runtime para reproducir la semántica nativa de IBM i fuera del hardware original.
  • En desarrollo
    Pipeline end-to-end Encadenado completo RPG → ejecutable, con verificación de equivalencia.
  • Kotlin
